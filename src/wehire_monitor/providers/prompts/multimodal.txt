请从以下微信公众号招聘文章中提取招聘岗位信息。文章可能以纯文本、纯图片或文本+图片形式提供,请综合文本与图片中明确可见的内容进行抽取。

重点：
1. 保留岗位表格的行列对应关系,不要把相邻行的地点/岗位串错。
2. 邮箱、网址、报名方式必须逐字符准确识别;无法确认的字符用 ? 标记,并在 warnings 中说明。
3. 不要根据常识补全公司名、地点或截止日期;只抽取明确出现的信息。
4. 如果图片底部区域(可能包含邮箱、二维码、报名方式),请特别关注底部信息。
5. 一篇文章可能包含多个岗位,输出 jobs 数组。
6. 不抽取校招、实习、宣讲会、培训类信息。
7. 如果是报名链接或二维码,没有邮箱,也要保留 apply_channel。
8. 返回严格 JSON,不要 Markdown。

抽取字段：
- company_name: 公司名称
- job_name: 岗位名称
- location: 工作地点
- apply_channel: 投递邮箱、报名链接、二维码说明或其他投递方式
- email: 邮箱;必须逐字符准确输出
- email_chars: 邮箱字符数组,例如 ["h","r","@","x",".","c","o","m"]
- deadline: 截止日期,统一 YYYY-MM-DD;无明确年份时结合文章发布时间推断,并标记 inferred=true
- source_evidence: 每个字段对应的原文短证据
- confidence: 0-100

JSON Schema:
{
  "article_type": "social_recruitment | campus_recruitment | internship | non_recruitment | unknown",
  "jobs": [
    {
      "company_name": "string|null",
      "job_name": "string|null",
      "location": "string|null",
      "apply_channel": "string|null",
      "email": "string|null",
      "email_chars": ["string"],
      "deadline": {"date": "YYYY-MM-DD|null", "inferred": false},
      "source_evidence": {},
      "confidence": 0
    }
  ],
  "warnings": []
}

说明：
- article_type 取值: social_recruitment(社招) / campus_recruitment(校招) / internship(实习) / non_recruitment(非招聘) / unknown(无法判断)
- confidence 为 0-100 整数
- email_chars 是邮箱分段字符列表(如 "h-r@" 拆为 ["h", "-", "r", "@"]),不要返回字符串
- deadline 必须是对象 {"date": "...", "inferred": false},不要返回字符串
- 发布时间参考: {{publish_time}}(用于推断截止日期年份)

文章标题: {{title}}
